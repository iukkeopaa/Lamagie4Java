### 创建对象所用的运算符



在 Java 中，主要使用 `new` 运算符来创建对象。`new` 运算符的作用是在堆内存中为对象分配内存空间，并调用对象的构造方法来初始化对象。以下是使用 `new` 运算符创建对象的示例：





```java
class Person {
    String name;
    int age;

    // 构造方法
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

public class Main {
    public static void main(String[] args) {
        // 使用 new 运算符创建 Person 对象
        Person person = new Person("Alice", 25);
    }
}
```



在上述代码中，`new Person("Alice", 25)` 语句通过 `new` 运算符在堆内存中创建了一个 `Person` 对象，并调用了 `Person` 类的构造方法进行初始化。然后将该对象的引用赋值给 `person` 变量。

### 对象实体与对象引用的区别

#### 概念



- **对象实体**：也称为对象实例，是在堆内存中实际分配的一块内存区域，用于存储对象的属性值等数据。对象实体包含了对象的具体状态和行为实现，是对象的实际存在形式。
- **对象引用**：是一个变量，它存储的是对象实体在堆内存中的内存地址。通过对象引用，我们可以访问和操作对象实体。对象引用本身存储在栈内存中。

#### 内存存储位置



- **对象实体**：存储在堆内存中。堆内存用于存储所有通过 `new` 运算符创建的对象，它是程序运行时动态分配内存的区域。
- **对象引用**：存储在栈内存中。栈内存主要用于存储局部变量，包括对象引用变量。当方法被调用时，会在栈上为方法的局部变量分配内存空间，方法调用结束后，这些局部变量的内存会被释放。

#### 生命周期



- **对象实体**：对象实体的生命周期从使用 `new` 运算符创建时开始，直到没有任何引用指向它，并且被垃圾回收器回收为止。垃圾回收器会定期检查堆内存中不再被引用的对象，并回收它们所占用的内存空间。
- **对象引用**：对象引用的生命周期取决于它所在的作用域。当对象引用所在的方法执行结束，或者对象引用所在的代码块执行完毕，对象引用的生命周期就结束了。此时，对象引用所占用的栈内存会被释放，但对象实体本身可能仍然存在于堆内存中，直到没有其他引用指向它。

#### 操作方式



- **对象实体**：不能直接操作对象实体，必须通过对象引用间接访问和操作对象实体的属性和方法。
- **对象引用**：可以通过对象引用调用对象实体的方法、访问和修改对象实体的属性。例如：









```java
class Car {
    String color;

    public void drive() {
        System.out.println("汽车正在行驶");
    }
}

public class Main {
    public static void main(String[] args) {
        // 创建 Car 对象并获取引用
        Car car = new Car();
        // 通过引用访问对象实体的属性
        car.color = "红色";
        // 通过引用调用对象实体的方法
        car.drive();
    }
}
```







在上述代码中，`car` 是对象引用，通过 `car` 可以访问和修改 `Car` 对象实体的 `color` 属性，也可以调用 `Car` 对象实体的 `drive` 方法。
