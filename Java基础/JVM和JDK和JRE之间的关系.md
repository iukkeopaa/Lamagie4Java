### **1. JVM（Java Virtual Machine，Java 虚拟机）**

#### 定义

- JVM 是一个虚拟的计算机，它负责执行 Java 字节码（`.class` 文件）。
- JVM 是 Java 实现“一次编写，到处运行”的核心。

#### 主要功能

- **加载字节码**：将 `.class` 文件加载到内存中。
- **解释/编译字节码**：将字节码解释或编译为机器码执行。
- **内存管理**：包括堆、栈、方法区等内存区域的管理。
- **垃圾回收**：自动回收不再使用的对象，释放内存。

#### 特点

- JVM 是平台相关的，不同操作系统需要安装对应的 JVM 实现。
- JVM 本身不包含开发工具或类库。

------

### **2. JRE（Java Runtime Environment，Java 运行时环境）**

#### 定义

- JRE 是运行 Java 程序所需的最小环境，包含了 JVM 和运行 Java 程序所需的核心类库（如 `java.lang`、`java.util` 等）。

#### 主要组件

- **JVM**：用于执行 Java 字节码。
- **核心类库**：提供 Java 程序运行所需的基础类和方法。

#### 特点

- JRE 只能运行 Java 程序，不能用于开发。
- 如果只需要运行 Java 程序，安装 JRE 即可。

------

### **3. JDK（Java Development Kit，Java 开发工具包）**

#### 定义

- JDK 是用于开发 Java 程序的完整工具包，包含了 JRE 和开发工具。

#### 主要组件

- **JRE**：用于运行 Java 程序。
- **开发工具**：
    - `javac`：Java 编译器，将 `.java` 文件编译为 `.class` 文件。
    - `java`：Java 解释器，用于运行 Java 程序。
    - `javadoc`：生成 API 文档。
    - `jar`：打包工具，用于创建 JAR 文件。
    - `jdb`：Java 调试器。
- **额外类库**：开发工具和扩展类库（如 `javax` 包）。

#### 特点

- JDK 是开发 Java 程序的必备工具。
- 如果需要进行 Java 开发，必须安装 JDK。

------

### **三者关系**

1. **JVM** 是运行 Java 程序的核心，但单独使用 JVM 无法运行 Java 程序，因为它缺少核心类库。
2. **JRE** 包含了 JVM 和核心类库，可以运行 Java 程序，但不能用于开发。
3. **JDK** 包含了 JRE 和开发工具，既可以运行 Java 程序，也可以开发 Java 程序。

#### 关系图

复制

```
JDK = JRE + 开发工具
JRE = JVM + 核心类库
```

------

### **使用场景**

- **JVM**：
    - 用于执行 Java 字节码。
    - 是 Java 跨平台的基础。
- **JRE**：
    - 用于运行 Java 程序。
    - 适合不需要开发的用户。
- **JDK**：
    - 用于开发 Java 程序。
    - 适合开发者。

------

### **安装选择**

- 如果只需要运行 Java 程序，安装 **JRE** 即可。
- 如果需要开发 Java 程序，必须安装 **JDK**。

------

### **总结**

- **JVM** 是 Java 程序运行的核心。
- **JRE** 是运行 Java 程序的最小环境。
- **JDK** 是开发 Java 程序的完整工具包。
